# SpeakMate 项目演示讲解稿

---

## 一、开场介绍（1 分钟）

### 项目概述

> "大家好，这是 **SpeakMate**，一款 AI 英语口语陪练应用。"
>
> "它的核心价值是：**为英语学习者提供沉浸式场景对话练习**，用户可以随时随地和 AI 进行英语对话，并获得实时纠错、发音指导和课后总结。"

### 技术亮点

> "- 前端：React 18 + TypeScript + Vite + Tailwind CSS + Zustand 状态管理"
> "- 后端：FastAPI + SQLAlchemy 异步 + SQLite + Alembic 数据库迁移"
> "- AI 引擎：支持 DeepSeek / Qwen / OpenAI 多厂商 LLM，自动降级兜底"
> "- 语音能力：Whisper 语音识别 + Web Speech API 语音合成"
> "- 亮点功能：**课后总结、三档难度差异化、音频级发音评分、PDF 报告导出**"

```

cd backend
uvicorn app.main:app --host 127.0.0.1 --port 8000

cd frontend
```

打开浏览器 → http://localhost:5173

>

### 步骤 3：PracticeRoom 练习室（3 分钟）

> "进入练习室，页面分为三栏布局："
>
> "- **左侧面板**：显示场景信息、角色、难度标签（Easy · 中等），以及暂停/重置按钮"
> "- **中间面板**：对话区 + 输入区 + 麦克风按钮 + 语音播报开关"
> "- **右侧面板**：实时反馈——发音环、语法环、流利度环、纠错面板、发音详情、词汇统计"

#### 功能 1：对话 + 语音播报

> "AI 的开场白是英文的，并且会自动语音朗读。顶栏的喇叭开关可以控制语音播报。"

#### 功能 2：发送消息

> "我在输入框输入英文，按回车发送。AI 会流式返回回复，同时右侧面板实时更新纠错和发音分析。"

**对话脚本（面试 - 中等难度）：**

| 轮次 | 你发送 | AI 会回复什么 |
|------|--------|-------------|
| 1 | （AI 开场） | Hi there, welcome! Tell me a little about yourself and your background. |
| 2 | I am a software engineer with three years of experience. I like coding and solve problems. | That's great! What kind of projects have you worked on recently? |
| 3 | I work on a web application using React and Python. My team have five peoples. | Interesting! （纠错：have→has, peoples→people）So you're full-stack then. What's the most challenging part of your current project? |
| 4 | The most challenging part is manage the big data. We need to make the query more faster. | I see — managing large datasets can be tricky. （纠错：manage→managing, make faster→speed up）How do you handle tight deadlines? |
| 5 | We use agile. We have daily meeting every morning and sprint planning every two week. | Sounds like a solid process. （纠错：meeting→meetings, week→weeks）One last question — where do you see yourself in three years? |

> "注意观察——每个 AI 回复都会自动朗读，右侧面板实时显示纠错内容。"

#### 功能 3：暂停会话

> "点击左侧的「暂停会话」按钮——对话区出现毛玻璃遮罩，输入栏隐藏，状态变为「已暂停」。再点「继续会话」恢复。"

#### 功能 4：重置会话

> "点击「重置会话」——弹出确认框，确认后重新开始对话。"

#### 功能 5：麦克风语音输入

> "点击麦克风按钮开始录音，说完话再点一次结束录音，Whisper 自动转写为文字并发送。"

## 三、各场景完整对话示例

### 场景 1：面试（Interview）

```
你：I am a software engineer. I have work in this field for five years.
AI：（纠错：work→worked）That's solid experience. What would you say is your greatest strength as an engineer?
你：I think my strength is I can learn new technology quickly and I always finish task on time.
AI：（纠错：learn→pick up, task→tasks）Quick learner and reliable — those are great qualities. Can you tell me about a time you had to solve a difficult problem under pressure?
```

### 场景 2：餐厅（Restaurant）

```
你：I want to order some food. What do you recommend?
AI：Oh you'll love the grilled salmon — it's our most popular dish. Served with roasted vegetables and lemon butter. Would you like to try that?
你：Sounds good. I also want a salad. How much is the Caesar salad?
AI：Great choice! The Caesar is fourteen dollars. So that's one grilled salmon and one Caesar salad — anything to drink with that?
```

### 场景 3：会议（Meeting）

```
你：I finished the frontend part of the project. The login page and dashboard is done.
AI：（纠错：is→are）Nice work, that's great progress. Are there any blockers or do you need more resources for the backend integration?
你：No blockers. But I think we need more time for testing. The deadline is too tight.
AI：Fair point. I'll talk to the client about extending the deadline by a week. Can you update the project timeline doc by Tuesday?
```

### 场景 4：旅行（Travel）

```
你：I want to travel to Japan next month. Can you help me find a flight?
AI：Oh Japan is amazing! You'll have a fantastic time. Let me check — morning flights are usually quieter. Are you flexible with dates or do you have specific days in mind?
你：I want to leave on June 15 and come back on June 22. I prefer morning flights.
AI：Perfect, let me pull up the options. There's CA1234 departing at eight in the morning, or MU5678 at ten thirty. The earlier one is usually cheaper — want me to book that one?
```

## 五、演示时的技术问答

**Q: 如果没有 LLM 怎么办？**
> "后端内置了降级脚本，每个场景有预置对话模板。即使 API Key 不可用，对话也能正常进行。"

**Q: 数据库怎么管理？**
> "使用 Alembic 做数据库迁移，表结构版本控制。启动时自动运行 migration，保证数据库与代码同步。"

**Q: 为什么选择这几个场景？**
> "面试、餐厅、会议、旅行是最常见的生活/工作场景，覆盖了不同语境和词汇量。未来可以扩展更多场景。"

**Q: 难度是怎么体现的？**
> "三个层面：① 开场白从短句到长难句 ② System Prompt 控制 LLM 词汇和句式复杂度 ③ 简单模式有中文翻译辅助理解。"
